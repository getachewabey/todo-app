## Setup Instructions

### 1. Clone & Install Dependencies

```bash
# Install frontend dependencies
npm install

# Install python dependencies for deployment wrapper
pip install streamlit
```

### 2. Supabase Configuration

1.  Create a new Supabase project.
2.  Run the following SQL in your Supabase SQL Editor to create the `tasks` table:

    ```sql
    create table tasks (
      id bigint generated by default as identity primary key,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null,
      text text not null,
      category text not null,
      completed boolean default false,
      user_id uuid references auth.users default auth.uid()
    );
    ```

3.  Disable **Confirm Email** in Authentication > Providers > Email (for easier testing).
4.  Create a `.env` file in the root directory:

    ```env
    VITE_SUPABASE_URL=your_supabase_url
    VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
    ```

### 3. Run Locally (Development)

To run the React app with hot reloading:

```bash
npm run dev
```

Visit `http://localhost:5173`.

### 4. Run Deployment Wrapper (Streamlit)

To test the deployment version that serves the built static files:

```bash
# Build the React app
npm run build

# Run the Streamlit wrapper
streamlit run streamlit_app.py
```

Visit `http://localhost:8501`.

## ðŸš€ Deploying to Streamlit Cloud

1.  **Push to GitHub**:
    *   Make sure to commit the `dist` folder (it is un-ignored for this purpose).
2.  **New App**: Go to [share.streamlit.io](https://share.streamlit.io/) and create a new app.
3.  **Settings**:
    *   **Repository**: Select your repo.
    *   **Main file path**: `streamlit_app.py`.
4.  **Secrets (Crucial)**:
    *   In Streamlit Cloud settings, go to **Secrets**.
    *   Add your Supabase credentials (copy from `.env`):
    
    ```toml
    VITE_SUPABASE_URL = "your_url"
    VITE_SUPABASE_ANON_KEY = "your_key"
    ```
    *(Note: The React app uses baked-in env vars from build time, but adding them here is good practice if we ever move to server-side logic).*

## Project Structure
